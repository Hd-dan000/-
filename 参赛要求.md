| 赛题简介：介绍整个赛题的实现目标、实用价值、涉及技术和整体要求                  | 本赛题聚焦实训教学评价管理。围绕大模型技术的落地应用，构建一套覆盖软件实训全流程的自动化、智能化检查评价体系。具备显著的教学实用价值。对教师而言，大幅减轻批量批改代码、审核文档的工作量，规避人工评价的主观偏差，提升实训检查效率与评价客观性;对学校而言，汇聚实训全量数据形成教学质量画像，为课程体系优化、实训方案调整提供数据支撑;对学生而言，获得即时性评价反馈与薄弱点分析，助力针对性补漏，强化实训效果与自主学习能力。                                                                                                                                                                                                     |
| ------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 赛题业务场景：描述赛题相关的真实企业业务背景。从真实场景中，适当简化或者提炼出适合比赛的赛题场景 | 当前职业教育机构及高校校企合作项目中，普遍存在软件实训成果评估效率低、标准不统一的痛点。企业内训或实训基地需批量审核学员代码、项目文档，传统人工方式耗时久、主观偏差大，且难以快速形成数据化评估报告供复盘优化;高校与企业协同实训时，也需一套兼顾教学要求与企业标准的评价工具，打通实训成果与岗位需求的衔接。基于此，本赛题聚焦“高校-企业协同实训”核心场景，打造智能化实训评价系统。基于上述业务背景，聚焦教育行业软件实训教学，参赛者可基于本地大模型或公有云大模型API，开发覆盖实训成果上传、智能核查、多维度评价、报表生成与导出(支持Excel/PDF格式)、评价标准自定义等核心场景。                                                                                                                   |
| 基本功能需求                                           | 采用B/S架构开发，软件需部署在自主指令系统LoongArch架构+麒麟高级服务器版上运行(不满足该要求视为0分)。功能要求：1、支持本地大模型服务部署或调用云端大模型服务，具备 PC Web 或 App 可视化界面，方便用户访问与操作；2、实训成果上传与解析：支持文档(Word/PDF)、报告、软件界面操作截图等多格式成果上传，结合大模型技术自动解析文件内容，提取核心信息用于后续核查与评价；3、智能核查功能：实现实训要求与实训效果校验、逻辑漏洞初步识别，同时核查实训步骤完整性，减少人工排查成本；4、多维度评价管理：可支持自定义评价指标(如代码质量、文档规范性、功能实现度等)及权重分配，结合大模型技术能力给出客观评分，同时预留教师主观评分入口，兼顾智能化与灵活性；5、报表生成与导出：生成实训评价报告、科目实训数据统计报表，支持Excel/PDF格式导出，报表含可视化图表，直观呈现评价结果。<br> |
