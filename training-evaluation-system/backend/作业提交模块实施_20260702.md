# 作业提交模块实施记录

## 完成内容

### 1. 数据库 - `submit_work` 表创建
- **表名**: `submit_work`（MySQL「信息」库）
- **字段**: id, student_id, student_no, student_name, class_id, class_name, training_id, training_title, file_path, file_original_name, file_extension, file_size, content_type, status, teacher_score, teacher_comment, submitted_at, reviewed_at
- **索引**: 按 student_id, class_id, training_id, (class_id, training_id) 建立索引
- **执行脚本**: `backend/create_submit_work.py`

### 2. 后端 API — `routers/homework.py`
| API | 方法 | 说明 |
|-----|------|------|
| `/api/homework/upload` | POST | 学生上传作业（multipart/form-data） |
| `/api/homework/teacher/submissions` | GET | 教师查询本班学生作业（自动按登录教师ID过滤授课班级） |
| `/api/homework/teacher/class-list` | GET | 教师获取授课班级列表（含学生数） |
| `/api/homework/teacher/trainings` | GET | 教师获取有提交记录的实训任务列表 |
| `/api/homework/review/<id>` | PUT | 教师评分评语（0-100分） |
| `/api/homework/file/<id>` | GET | 文件在线预览/下载（PDF内嵌预览，MP4视频播放） |

### 3. 学生端同步
- `routers/training.py` 中 `handle_submit_work` 已注入 `_sync_to_submit_work` 函数
- 学生通过原有 `/api/training/<id>/submit` 提交作业后，自动同步写入 MySQL `submit_work` 表
- 文件在 `upload/` 目录和 `upload/homework/` 目录各存一份

### 4. 配置更新 — `config.py`
- 新增 `UPLOAD_HOMEWORK_DIR`：指向项目根 `upload/homework/`
- 新增 `ALLOWED_HOMEWORK_EXTENSIONS`：`.pdf` / `.doc` / `.docx` / `.mp4`
- 新增 `MAX_HOMEWORK_SIZE`：100MB

### 5. 教师端页面 — `frontend/teacher_homework.html`
- 我的班级（班级列表 + 学生数统计）
- 作业提交管理（按班级/实训/状态筛选）
- 文件预览（PDF内嵌 / MP4播放 / 其他格式下载）
- 评分弹窗 + 已评分详情查看

### 6. 路由注册 — `main.py`
- 所有作业 API 路由已注册到 `RequestHandler.route_request()`
- `homework` 模块已导入
