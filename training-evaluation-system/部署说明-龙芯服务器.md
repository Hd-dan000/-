# 实训教学检查评价系统部署说明（龙芯服务器版）

## 1. 部署目标

本项目采用前后端分离方式部署，前端由 Nginx 直接托管静态资源，后端由 Python 原生 HTTP 服务提供 API。系统可运行在 LoongArch 架构的银河麒麟高级服务器环境中。

## 2. 运行环境

- 操作系统：银河麒麟高级服务器 V10 / V11
- CPU 架构：LoongArch
- 内存：8GB 及以上
- 磁盘：256GB 及以上
- 数据库：MySQL 8.0 或兼容版本，用于用户、教师、学生信息
- 运行时：Python 3.10+、Node.js 18+、Nginx

## 3. 项目结构说明

- `backend/`：后端服务，启动入口为 `main.py`
- `frontend/`：Vue 3 前端项目，构建后输出到 `frontend/dist`
- `deploy/`：部署脚本和 Nginx 配置

## 4. 部署步骤

### 4.1 准备代码

将整个项目目录复制到服务器，例如：`/opt/training-evaluation`。

### 4.2 配置数据库

1. 创建 MySQL 数据库并执行 `backend/init_info_db.sql`，生成 `teachers`、`admins`、`students` 等表。
2. 执行 `python backend/init_admin.py` 创建默认管理员账号。
3. 确认 `backend/config.py` 中的 MySQL 配置正确，或通过环境变量覆盖：
   - `MYSQL_HOST`
   - `MYSQL_PORT`
   - `MYSQL_USER`
   - `MYSQL_PASSWORD`
   - `MYSQL_DATABASE`

### 4.3 配置大模型服务

系统支持两种方式：

- 云端 API：配置 `LLM_PROVIDER=openai`，填写 `LLM_API_BASE`、`LLM_API_KEY`、`LLM_MODEL`
- 本地模型：配置 `LLM_PROVIDER=ollama`，填写 `OLLAMA_HOST`、`OLLAMA_MODEL`

### 4.4 安装依赖与构建前端

当前后端只保留了一个第三方 Python 依赖：PyMySQL。为了尽量减少龙芯服务器上的兼容性风险，建议优先使用系统仓库里的 `python3-pymysql`，如果系统仓库没有，再让脚本通过 pip 兜底安装。

在项目根目录执行：

```bash
bash deploy/deploy.sh
```

脚本会自动完成：

- Python 虚拟环境创建
- 后端依赖安装
- 前端依赖安装
- 前端静态资源构建
- systemd 服务注册
- Nginx 配置参考

如果需要手动执行，可按以下顺序：

```bash
cd /opt/training-evaluation/backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 如果系统已经安装了 python3-pymysql，这一步可以跳过

cd ../frontend
npm install
npm run build
```

### 4.5 启动后端

后端运行命令：

```bash
cd /opt/training-evaluation/backend
source ../venv/bin/activate
python main.py
```

建议使用 systemd 常驻运行，脚本会创建 `training-eval.service`。

### 4.6 配置 Nginx

将 `deploy/nginx.conf` 作为参考，核心逻辑如下：

- `/` 指向 `frontend/dist`
- `/api/` 反向代理到 `127.0.0.1:8000`
- `/uploads/` 和 `/reports/` 反向代理到后端

修改后执行：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

## 5. 功能配置建议

- 文件上传限制：进入“系统基础配置”页面调整最大上传大小
- 报表模板：进入“系统基础配置”页面选择标准或精简模板
- 系统开关：可按需关闭提交、评价、补评、报表生成
- 账号权限：在“账号权限管理”中维护学生、教师、管理员账号
- 运维中心：在“统计与日志”页面查看跨班级统计、操作日志并下载备份包

## 6. 备份与恢复

系统提供“统计与日志”页面的备份包下载功能，备份包包含：

- 当前 SQLite 数据库快照
- 操作日志 JSON
- 备份清单文件

恢复时可将数据库快照替换回原数据库文件，并重启服务。

## 7. 常用检查命令

```bash
sudo systemctl status training-eval
sudo journalctl -u training-eval -f
sudo nginx -t
```

## 8. 说明

当前项目后端采用 Python 原生 `http.server` 实现，不依赖 FastAPI/uvicorn；部署脚本已经按这一点调整。
